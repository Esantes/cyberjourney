// 01 — FEATURES

Every signal you need. None of the noise.

Built by people who hated logging into four sites to remember what they did last week. CyberJourney is the missing layer between you and your favourite platforms.

// 01

Unified progress

OverTheWire, TryHackMe, Hack The Box, PortSwigger — one source of truth. Manual entry, all yours, nothing leaves the browser.

// 02

XP & level system

Real difficulty-weighted XP, not participation trophies. Levels gate badges; badges gate bragging rights.

// 03

Markdown notes

Per-challenge notes with syntax highlighting, tag-based skill tracking, and full-text search across everything you've ever written.

// 04

Streaks & activity

An activity heatmap that doesn't lie. Daily streaks, weekly velocity, and "where did the week go" honesty.

// 05

Skill graph

Tags on every challenge roll up into a skill maturity view. See exactly which areas are carrying you and which need a Saturday.

// 06

Built-in terminal

Practice commands in a sandboxed terminal pane. Save snippets to your cheat sheet with one keystroke.

// 07

Pomodoro timer

Focus sessions that log themselves to your activity feed. Default 25/5, fully tweakable.

// 08

Achievements

50+ badges from "First Blood" to "100 Days Hardcore." Animated popups that feel earned, not patronising.

// 09

Export everything

One click to Markdown, JSON, or CSV. Your journey, your data. Local-first by design; cloud sync only when you ask.

// 02 — PRODUCT

Designed by someone who actually grinds the boxes.

Every screen is built around three principles: density without clutter, motion that means something, and zero friction between "I had an idea" and "it's logged."

Per-platform deep dives

OverTheWire Bandit levels 0–33 ship pre-seeded. Check off, take notes, move on.

Markdown-native journal

Categories for Linux, Networking, Web, Crypto, Privesc. Search across everything.

Activity heatmap

A year of streaks at a glance. Hover any day for that day's logs.

Animated achievements

Toast-style unlocks that won't trash your focus. Mutable in Settings.

// 04 — WHY I BUILT THIS

Honestly? Because nothing else fit.

I'm self-taught. I spend evenings on OverTheWire and weekends on Hack The Box. The platforms are great. What didn't exist was a place to see all of it together — my notes, my streaks, my skill drift. So I made one.

// REASON 01

For my own study

A dashboard I actually open every day. Streaks I can see. Notes I can search. Skills that update themselves from the tags I already use.

// REASON 02

To practice React seriously

A real app forces real decisions: state shape, component tiers, accessibility, motion, mobile-first. Tutorials don't push those buttons. This project does.

// REASON 03

As a portfolio piece

Something a recruiter can click, not a screenshot in a PDF. Local-first, modular, no fake backend smoke. The code is the resume.

// WHAT IT IS
  • +A single-page React app, one HTML entry, modular JSX
  • +Local-first — your data lives in localStorage
  • +Free, no signup, no telemetry, no servers
  • +Bandit levels 0–33 pre-seeded, sample THM / HTB / PSW too
  • +Built and maintained by one person, for fun
// WHAT IT ISN'T
  • –A SaaS. There's no plan, no billing, nothing to sell.
  • –A backend. No accounts, no database, no server to host.
  • –A team project. It's me, on weekends, learning.
  • –Polished forever. Some corners are intentionally rough.
  • –For everyone. Built to fit how I study.
// 05 — STACK

Made with deliberately boring tools.

No build server, no Docker, no microservices. Just the smallest stack that does the job well and keeps the focus on UI and architecture.

// FRONTEND
  • React 18 · functional, hooks-only
  • Vite · zero-config dev server
  • TailwindCSS + token CSS
  • Hash router · no library needed
// STATE & DATA
  • Context + reducer · one store
  • localStorage · the whole DB
  • Storage adapter · swap-ready
  • Append-only events · for the heatmap
// CRAFT
  • CSS transitions · every motion
  • Prefers-reduced-motion · honored
  • Mobile-first · 360px up
  • No emoji · just type and color
// WHAT I LEARNED BUILDING THIS
01State management without the dogma — Context + a single reducer covers more than people admit.
02A storage adapter from day one. Even if I never swap it, the discipline pays.
03CSS variables > theme objects when you want runtime tweaks to feel instant.
04Motion is a tool, not a vibe — one glow per screen, and only on state change.
05Mobile-first really does mean first. Designing at 1280 makes you lie to yourself.
06The thing I actually use every week is the markdown journal. Build for yourself first.
